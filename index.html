Adding Images to Your HTML

There are a couple of reasons for adding images to your HTML. One reason is that you would like to add an image for decoration. The other reason is because you have images that are necessary content, and if you remove them from the page the user would be missing needed information.

Images have two required attributes: src, which tells the browser where the image is and should use a relative link, and alt, which uses text to describe the image so that you could imagine what it looks like if you can't see it. Remember that if your image has any text in it, you have to include that in the alt text. It is also important to add the height and width attributes to help the page layout while loading.

✱ Adding Decorative Images to HTML

Decorative images should have an empty alt attribute so that screen readers skip them. You can't leave the attribute off completely — it must be empty.

✱ Adding Images as Content in HTML

For images that are content, you must use descriptive alt text and the correct width and height.

Adding Video to Your HTML

Many sites today use video, as it is something audiences prefer. You can use multiple sources to allow the browser to play the one it supports best. You should use relative links for your local videos.

Adding a Favicon to Your Web Page

A favicon helps users recognize your site in their browser tabs or bookmarks. You can use favicon.io or another generator to create one.

Using SVGs in HTML

SVGs are scalable vector graphics that look sharp on any screen size. They can be embedded directly into HTML or linked like normal images. We'll use them in the footer below.

Using Tables in HTML

Tables are used for structured data, such as schedules. Each table should have a <caption> and use scope attributes for accessibility.
